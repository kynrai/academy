# Week 6 – Data Structures and Modelling

| Day | Topic                                   |
| --- | --------------------------------------- |
| 1   | Arrays and Lists                        |
| 2   | Maps and Dictionaries                   |
| 3   | Structs, Classes, and Interfaces        |
| 4   | Generics and Type Safety                |
| 5   | Project – Typed Inventory System        |

## Objectives

- Work confidently with collections in Python, TypeScript, C#, and Go.
- Model real-world data as structs, classes, and interfaces.
- Understand when to choose a list versus a map.
- Apply generics to write reusable, type-safe code.

## Topics

- Lists and slices: creation, indexing, appending, slicing, and sorting.
- Stacks and queues as list patterns.
- Dictionaries and maps: creation, lookup, deletion, and iteration.
- Grouping and counting with maps.
- Defining custom types: Python dataclasses, TypeScript interfaces and classes, C# records and classes, Go structs.
- Methods on types and constructor/factory patterns.
- Interfaces and polymorphism in each language.
- Generics: type parameters, constraints, and practical use cases.

## Hands-On Activities

- Build a sorted product list with filtering in each language.
- Group products by category using a map and count totals.
- Define a `Product` type with validation in all four languages.
- Implement a generic stack and observe how each language expresses type parameters.
- Build a typed inventory system as the week's culminating project.

## Deliverables

- A `Product` type definition in each language with at least three methods.
- A typed inventory system that supports add, remove, search, and group-by-category.

## Assessment

- Code review: type correctness, interface usage, and appropriate collection choice.
