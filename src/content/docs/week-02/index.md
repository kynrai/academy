## Overview

Day 1 gets all four language runtimes installed and running, builds a Hello World CLI in each, and explains how command-line interfaces work. Day 2 introduces HTTP and Server-Side Rendering, Day 3 covers REST APIs in all four languages, and Days 4–5 build a complete server-rendered web app — Bibliotheca, a book catalog with templates, search, forms, and validation — that uses no client-side framework.

By the end of the week you can install any of the four languages, build a CLI that handles flags and arguments correctly, build a JSON API, and build a multi-page SSR web app with forms.

## What you will learn

| Day | Topic |
| --- | ----- |
| 1   | Install all four runtimes; build a Hello World CLI in each; learn how flags and arguments work |
| 2   | HTTP basics, how browsers work, and Server-Side Rendering in all four languages |
| 3   | REST APIs in all four languages |
| 4   | SSR vs CSR through HTTP; start the Bibliotheca project (home, list, detail) |
| 5   | Forms, validation, redirect-after-POST; complete Bibliotheca |
| 6   | Weekend challenges |

## The four languages

| Language | Runtime | Package manager | Key file | Output |
| -------- | ------- | --------------- | -------- | ------ |
| Python | CPython | uv | `pyproject.toml` | Interpreted script |
| JavaScript | Node.js | npm | `package.json` | Interpreted script |
| C# | .NET | NuGet / dotnet | `.csproj` | Compiled (JIT) |
| Go | Go toolchain | go modules | `go.mod` | Native binary |
