# Week 2 – Language Setup, Foundations, and Web Development

## Overview

Day 1 gets all four language runtimes installed and running, builds a Hello World CLI in each, and explains how command-line interfaces work. Days 2–5 cover the web layer in depth: HTTP fundamentals and Server-Side Rendering, REST APIs, Client-Side Rendering, and finally a full-stack project that combines all three patterns in one server.

By the end of the week you can install any of the four languages, build a CLI tool that handles flags and arguments correctly, build a JSON API, and serve both server-rendered HTML and client-side rendered content.

## What you will learn

| Day | Topic |
| --- | ----- |
| 1   | Install all four runtimes; build a Hello World CLI in each; learn how flags and arguments work |
| 2   | HTTP basics, how browsers work, and Server-Side Rendering in all four languages |
| 3   | REST APIs in all four languages |
| 4   | Client-Side Rendering with JavaScript |
| 5   | Full-Stack Project – SSR, REST API, and CSR in one server |
| 6   | Weekend challenges |

## The four languages

| Language | Runtime | Package manager | Key file | Output |
| -------- | ------- | --------------- | -------- | ------ |
| Python | CPython | uv | `pyproject.toml` | Interpreted script |
| JavaScript | Node.js | npm | `package.json` | Interpreted script |
| C# | .NET | NuGet / dotnet | `.csproj` | Compiled (JIT) |
| Go | Go toolchain | go modules | `go.mod` | Native binary |
